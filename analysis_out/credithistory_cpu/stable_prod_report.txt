--- [CRITICAL] Idle in transaction / нет таймаута ---
  Finding: sessions.high_idle_in_transaction, sessions.idle_timeout_disabled | stable 2/2 (100%) | reports: prom1, prom2
  Example: credithistory: idle_in_transaction_time=13277.7s (threshold 3600s), idle_in_transaction_session_timeout=0
  Problem: Долгие idle in transaction сессии держат snapshots, блокируют vacuum
  Problem: и увеличивают bloat — одна из самых частых рекомендаций коммьюнити.
  GUC tuning | change safety: ОСТОРОЖНО (тест + мониторинг) | potential impact: среднее
    • idle_in_transaction_session_timeout → set_nonzero | [ОСТОРОЖНО (тест + мониторинг)] [impact: среднее]
      Current: prom1=0, prom2=0
      Why: Стабильный idle in transaction без таймаута — классический production risk.
      Postgres Pro: Postgres Pro / PostgreSQL: 60s–300s типично для OLTP. Защита от утечки snapshots и bloat.
  Actions:
    - Исправить application: COMMIT/ROLLBACK, pool settings
    - Найти источник в pg_stat_activity
    - Установить idle_in_transaction_session_timeout
    - Исправить application connection pooling / missing COMMIT
    - Найти источник зависших транзакций в pg_stat_activity
    - Установить idle_in_transaction_session_timeout (например 60s–300s)
